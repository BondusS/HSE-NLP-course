# Отчет по домашнему заданию: Named Entity Recognition и сжатие BERT

## Выполненная задача
Дообучена модель BERT на датасете CoNLL-2003 для задачи распознавания именованных сущностей (NER)
и применены методы оптимизации для уменьшения размера модели до **менее 20 миллионов параметров**
при сохранении максимально возможного качества (по метрике F1-score).

## Задание 1: Выравнивание токенов с метками
При токенизации BERT использует WordPiece, который разбивает сложные слова на подтокены.
Чтобы обучить модель NER, нужно сопоставить каждому такому подтокену метку.

Для этого реализована функция `align_labels_with_tokens`:
1. Специальные токены (`[CLS]`, `[SEP]`) кодируются значением `-100`, чтобы лосс-функция их игнорировала.
2. Первый токен слова получает оригинальную метку.
3. Последующие подтокены (продолжения слова) получают либо ту же метку, либо, в случае меток `B-`,
меняются на соответствующие `I-`, чтобы сохранить корректность BIO-нотации.

## Задание 2: Дообучение BERT (Учитель)
В качестве базовой модели использован `bert-base-cased` (108M параметров). Это наша модель-учитель.

**Параметры обучения:**
- `learning_rate`: $2\cdot10^{-5}$
- `num_train_epochs`: 3
- `batch_size`: 16
- `weight_decay`: 0.01

**Результаты:**
- **Число параметров:** 107.7M
- **F1 на тесте:** **0.9018**

Модель достигла целевого показателя (>0.9), что позволяет использовать её логиты для качественной дистилляции.

## Задание 3: Факторизация матрицы эмбеддингов
Матрица эмбеддингов в BERT занимает около 22% всех параметров.
Мы применили метод из ALBERT — разложение матрицы $V \times H$ на две: $V \times E$ и $E \times H$.

**Метод:**
Использовано **SVD-разложение** с рангом **64**. Это позволило инициализировать новые слои не случайным шумом,
а математическим приближением весов учителя, что значительно ускорило дообучение.

**Результаты:**
- **Число параметров:** ~86.5M (уменьшение на ~21.2M)
- **F1 на тесте:** **0.8758**

Мы успешно преодолели порог в 20M сокращенных параметров, сохранив высокую метрику (потеря F1 составила всего ~0.026).

## Задание 4: Дистилляция знаний
Цель — создать "Ученика" размером менее 20M параметров.

**Конфигурация Ученика:**
- Количество слоев: **2** (вместо 12)
- Факторизация эмбеддингов: ранг **64**
- Общее число параметров: **16.48M**

**Метод дистилляции:**
Реализован кастомный `DistillationTrainer`. Функция потерь объединяет:
1. `Hard Loss`: CrossEntropy по истинным меткам.
2. `Soft Loss`: KL-дивергенция между логитами учителя и ученика с температурой $T=2.0$ и весом $\alpha=0.5$.

**Результаты:**
- **F1 на тесте:** **0.80+** (целевой порог преодолен).

Дистилляция позволила крошечной 2-слойной модели показывать результаты значительно выше, чем при обычном обучении с нуля.


## Задание (по выбору): Удаление голов внимания (Head Pruning)
Реализован подход на основе статьи *"Are Sixteen Heads Really Better than One?"*.
Метод позволяет определить избыточные головы внимания и физически удалить их из архитектуры.

**Методология:**
1. В механизм Attention добавлены обучаемые **гейты** (числа от 0 до 1), на которые множатся выходы каждой головы.
2. Введена **$L_1$-регуляризация** на значения гейтов ($\lambda=0.0001$). Модель штрафуется за "включенные" головы.
3. После обучения гейтов головы с весом ниже **0.0032** были удалены вручную, т.к. метод `model.prune_heads()` не работает.

**Результат:** Удалено **132 головы** (из 144).
Модель стала легче в вычислениях, но сильно потеряв F1 (**~0.34**).

## Задание (на выбор): Прунинг методом SparseGPT
Реализована техника пост-тренировочного прунинга весов, позволяющая добиться высокой разреженности (sparsity) без сложного дообучения.

**Методология:**
1. Использована небольшая выборка данных (128 примеров) для калибровки и сбора статистики активаций.
2. Для каждого линейного слоя рассчитывалась аппроксимация обратного Гессиана ошибки.
3. Веса с наименьшим вкладом в выходной сигнал слоя обнулялись,
а оставшиеся веса **корректировались**, чтобы минимизировать искажение выходного тензора слоя.

**Результат:** Достигнута разреженность **50%** во всех линейных слоях модели.
- **Sparsity:** 50% весов занулено.
- **F1 на тесте:** **0.8640**. Это значительно выше, чем при обычном удалении весов по магнитуде (Magnitude Pruning).
